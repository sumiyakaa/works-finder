# Works Finder リニューアル設計書

> **プロジェクト名：** AKASHIKI Works Finder v2
> **作成日：** 2026-04-08
> **ステータス：** 実装完了

---

## 1. プロジェクト概要

### コンセプト
**「わかりやすく見やすいのに、すごい技術が詰まっている」**

ポートフォリオ本体と同じブランド世界観を継承しつつ、Finderは「高級なツール」として機能性を全面に出す。
ポートフォリオとの関係は「同じ家の、別の部屋」。

### 技術スタック
- **フレームワーク：** Vite + React（完全新規）
- **デプロイ：** GitHub Pages
- **アニメーション：** Framer Motion
- **ルーティング：** React Router v6
- **PDF出力：** html2canvas + jsPDF

### ブランド統一要素（ポートフォリオから継承）
- フォント：Barlow / Noto Sans JP / IBM Plex Mono
- ロゴ：AKASHIKI SVGロゴ
- アクセントカラー：ゴールド系（操作UIに使用）

---

## 2. ページ構成（2階層）

### トップページ（一覧）
Heroなし。ページを開いた瞬間に一覧表示。ツール感重視。

**レイアウト：** 左サイドバー（常時表示・絞り込み）＋右メインエリア（ツールバー＋カードグリッド）

### 作品詳細ページ（/works-finder/:workId）
タブ切替（概要 / プレビュー / 技術詳細）＋前後ナビゲーション

---

## 3. ヘッダー

- 左：AKASHIKIロゴ（→ポートフォリオ本体へ）
- 中央〜右：ダークモード切替トグル
- 右：「制作について相談する」CTAボタン
- 詳細ページ時：ロゴ左に「← 一覧に戻る」追加

## 4. 左サイドバー（絞り込みUI）

### 配置
- PC（1024px+）：常時表示 240〜280px
- SP（〜1023px）：ドロワー方式（ボタンで開閉、左からスライド）

### 絞り込みカテゴリ（4種・アコーディオン式・チェックボックス複数選択・AND条件）

① **ジャンル（業種別）：** 飲食・カフェ / 美容・クリニック / 士業・コンサル / 不動産・住宅 / フィットネス / クリエイティブ・アート / ファッション・EC / 教育・スクール / テック・IT / その他

② **サイト種別：** LP / コーポレートサイト / ブランドサイト / Webツール・アプリ

③ **目的：** 集客・問い合わせ獲得 / ブランディング・認知 / ブランド訴求・販路拡張 / キャンペーン促進 / 採用

④ **技術タグ：** パララックス / スクロールアニメーション / CMS / フォーム実装 / マルチページ / 多言語対応 / ダークUI / CTA設計 / エディトリアルデザイン / アクセシビリティ / 見積シミュレーター / PDF出力

### サイドバーUI
- 選択中フィルターはサイドバー上部にタグ表示（×で個別解除）
- 「すべてクリア」ボタン
- 絞り込み結果件数リアルタイム表示

## 5. ツールバー（一覧上部）

左から：検索バー ｜ 表示切替（グリッド⇔リスト⇔サムネのみ）｜ 並び替え（新着順/Tier順/ページ数順）｜ 結果件数 ｜ 比較モードON/OFF

## 6. 作品カード

### グリッド表示（デフォルト）
- PC 3列 / タブレット 2列 / SP 1列
- カード内容：サムネイル＋タイトルのみ（ミニマル）
- ホバー時：クイックプレビュー（オーバーレイで概要・タグ・参考価格表示）

### リスト表示
- 左サムネ（小）＋中央タイトル・タグ・説明＋右参考価格。横幅100%。

### サムネのみモード
- PC 4〜5列 / タブレット 3列 / SP 2列。画像だけ大きく並べるギャラリー表示。

### 共通
- ホバー：scale(1.03) + shadow + 微チルト
- 比較モードON時：チェックボックス出現、2件選択で比較画面へ

### タブ3: 技術詳細

### PDF出力
- 「PDF出力」ボタンで概要＋技術詳細をPDF化（ブランドロゴ・フッター付き）

## 9. カラーモード

### ライトモード（デフォルト）
背景 #FAFAFA〜#FFFFFF / テキスト #1A1A1A / カード #FFFFFF / サイドバー #F5F5F5 / アクセント ゴールド系

### ダークモード

---

## 10. アニメーション仕様（優先度順）

1. **ページ遷移：** フェード＋スライドアップ/ダウン（Framer Motion AnimatePresence）
2. **カードホバー：** scale + shadow + 微チルト（0.3s ease）
3. **スクロール連動：** カードフェードイン（stagger 0.05s）
4. **サイドバー開閉（SP）：** 左スライド＋背景オーバーレイ

パフォーマンス：will-change適切使用 / GPU限定 / prefers-reduced-motion対応
