../Vanilla/ にあるAoki Design Studio Works FinderをReact + TypeScriptで完全再現する。
CLAUDE.mdの方針・品質基準・実装順序に厳密に従い、以下の手順で進めること。

品質を最優先とし、時間は一晩かかっても問題ない。
推測で進めず、判断に迷う箇所は必ずユーザーに確認を取ること。

---

## Step 1: Vanilla版の解析

@analyzer に ../Vanilla/ の完全な解析を依頼する。

解析結果を受け取り、以下を確認：
- Work型の全フィールドが網羅されているか
- render.js の関数 → コンポーネント対応が明確か
- 特殊処理（暗黙的なfeature追加、caseType導出等）が漏れていないか

---

## Step 2: React設計書の策定

@architect に解析結果を渡し、設計書の策定を依頼する。

設計書を受け取ったら、**ユーザーに提示して承認を待つ**。
ユーザーが承認するまで次のStepに進まないこと。

ユーザーに提示する際のフォーマット：
```
【設計書が完成しました】

以下の設計で実装を進めてよいか確認させてください。

- コンポーネント数: XX個
- カスタムフック数: XX個
- ユーティリティ関数: XX個
- 推定ファイル数: XX

[設計書の要点サマリー]

判断に迷った箇所:
1. [質問1]
2. [質問2]

承認いただければ実装を開始します。修正箇所があればお知らせください。
```

---

## Step 3: 段階的実装

ユーザー承認後、@implementer に設計書を渡し、Phase 1から順に実装を依頼する。

各Phase完了時に進捗を報告：
```
【Phase X 完了】
- 作成ファイル: [一覧]
- 型チェック: ✅ / ❌
- 次のPhase: [概要]
```

Phase 1-2（基盤層・状態管理層）完了後に一度ビルド確認を行う。
エラーがあればPhase 3に進む前に修正する。

Phase 3-6（コンポーネント実装）は連続で進める。

Phase 7（ルート統合）完了後にビルド確認を行う。

---

## Step 4: 品質レビュー

Phase 8として @reviewer に品質チェックを依頼する。

❌項目がある場合は @implementer に修正を指示。
全項目✅になるまでStep 3-4を繰り返す。

---

## Step 5: 最終報告

全工程完了後、以下のサマリーをユーザーに報告：

```
【移行完了】

■ 作成ファイル一覧
[ファイルパス: 行数]

■ 検証結果
- npm run build: ✅
- npx tsc --noEmit: ✅
- 機能一致: 13/13項目 ✅

■ 主要な設計判断
1. [判断内容]
2. [判断内容]

■ 確認のお願い
npm run dev で起動し、以下を目視確認してください：
- フィルタリング・検索・ソートの動作
- 比較機能（選択→パネル表示→表形式比較）
- 詳細モーダル（表示→フォーカストラップ→閉じ）
- CTAメモの動的切り替え
- レスポンシブ表示
```
